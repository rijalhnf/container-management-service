Alur Arsitektur Request

Secara garis besar, semua request akan melewati CORS middleware. Untuk endpoint di bawah /api, request juga melewati JWT middleware.

1) Client → Gin Router

Request masuk ke Gin dan dicocokkan dengan route.

2) CORS Middleware (global)

Menambahkan header CORS dan menangani preflight OPTIONS.

3) Public vs Protected Route

Public: /health, /swagger, /api/auth/login, /api/auth/register, /project-docs.

4) JWT AuthMiddleware (protected)

Validasi header Authorization: Bearer <token>. Inject claim (userID/email/role) ke context.

5) Controller

Bind JSON, jalankan validasi bisnis (contoh: ISO 6346), akses DB via config.DB.

6) GORM → PostgreSQL

Query/insert/update/delete. Beberapa endpoint menggunakan Preload() untuk eager loading relasi.

Tip: Referensi detail endpoint dan schema dapat dilihat pada Swagger UI: /swagger/index.html.

Validasi & Error Handling

Validasi dilakukan di beberapa layer: binding tags (request struct), middleware (JWT), dan validasi bisnis di controller (duplikasi & ISO 6346).

ISO 6346 — nomor container

Validasi format + check digit dilakukan oleh utils.IsValidISO6346(). 

Format: ^[A-Z]{3}[UJZ][0-9]{7}$
Check digit: sum(weighted) % 11, jika 10 => 0
Error (400): Invalid ISO 6346 container number

Duplikasi container number

Dicek pada batch payload dan dicek juga ke DB agar tidak ada container_number yang sama. 

Batch:
1) duplicate within payload => 400
2) existing in DB => 409

Single:
existing in DB => 409
Error (409): Container number already exists

Auth request validation (binding tags)

Validasi format email dan minimal password dilakukan lewat binding tags di models.RegisterRequest. 

RegisterRequest:
email: required,email
password: required,min=6

JWT AuthMiddleware

Endpoint protected berada di bawah group /api dan mewajibkan header Authorization. 

Authorization: Bearer <token>

Error:
- Authorization header is required
- Authorization header format must be: Bearer <token>
- Invalid or expired token

Unique constraints master data

Beberapa data master divalidasi unik di controller sebelum insert/update. 

ContainerType.Code => 409
Port.Code => 409
ShippingLine.Code => 409
Voyage.VoyageNumber => 409

Environment Variables

Docker Compose mendefinisikan env untuk container runtime. Di lokal, beberapa user menggunakan file core-project/.env.

Keamanan: jangan pernah mempublikasikan credential/secret mentah di dokumentasi. Gunakan placeholder seperti DB_PASSWORD=<redacted>.
Nama Fungsi
APP_PORT Port HTTP server (Docker: 8888).
DB_HOST, DB_PORT Host dan port PostgreSQL.
DB_USER, DB_PASSWORD, DB_NAME Kredensial dan nama database.
JWT_SECRET Secret untuk signing token JWT (wajib diganti di production).
CORS_ORIGIN Allowed origin untuk CORS (default *).
Dokumen ini dibuat untuk membantu onboarding dan debugging cepat. Untuk detail schema dan request body, gunakan Swagger UI.
